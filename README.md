# E-commerce-full-chain-personalized-recommendation-system
## 电商全链路个性化推荐系统
## 项目介绍
本项目基于淘宝亿级用户行为数据集（含百万级用户、200万+商品、超1亿条交互），搭建“特征工程 - 多路召回 - 粗排 - MMoE多任务精排 - 多样性重排”全链路电商个性化推荐系统，解决传统推荐系统商品马太效应严重、内容同质化、长尾商品曝光不足等核心业务痛点，实现百万级用户的个性化商品推荐服务。
## 数据集
UserBehavior是阿里巴巴提供的一个淘宝用户行为数据集，用于隐式反馈推荐问题的研究，用户行为包含用户ID，商品ID，商品类目ID，行为类型（包括点击、购买、加购、喜欢），时间戳，数据集链接[UserBehavior](https://tianchi.aliyun.com/dataset/649)
## 模型设计
- 特征工程体系搭建：针对电商用户转化链路，构建用户、商品、场景、交叉四大类共48+维核心特征，涵盖用户行为统计、历史转化偏好、商品转化效率、场景时段偏好、用户-商品交叉特征等维度；同时完成数据去重、噪声过滤、类型压缩等预处理，按时间维度划分训练、验证、测试集，避免数据泄露。
- 召回模块：设计6路差异化召回策略（热门召回、新品召回、ItemCF、UserTag 召回、DSSM召回、MIND多兴趣召回），覆盖用户不同消费动机；针对大规模向量检索效率问题，引入FAISS IVF快速索引并结合分块策略提升检索性能，通过单块用户分数矩阵量化各路召回对用户 - 物品的匹配度，最后加权融合完成TopN筛选。
- MMoE 多任务精排模型：实现CTR、CVR、加购率多任务联合优化；通过任务专属门控网络解耦多任务特征冲突，搭配轻量化专家网络与稀疏门控平衡精度与推理效率，引入动态加权损失解决样本不均衡问题。
- 针对传统精排的同质化、马太效应问题，构建“长尾保底 -品类打散 -DPP 多样性重排 -合规规则校验”四层重排架构：① 长尾保底层通过多层策略保证单用户Top10中≥3个长尾曝光；② 品类打散层结合品类计数与降权，控制单用户不同品类≥2，同品类≤5；③ DPP重排层结合CTR分与长尾加权设计质量分，基于商品特征相似度刻画差异，通过两阶段贪心选品兼顾长尾曝光与推荐多样性；④ 规则层先校验，再过滤、去重、不合规商品
## 实验结果
- 排序效果：CTR AUC达0.7071，加购率AUC达0.6676，相比LR、DeepFM、DIN等基线分别提升16.96%，9.22%，7.05%。
- 多样性、长尾效果：品类覆盖率72.24%，长尾商品整体曝光占比达38.48%,实现所有用户人均≥3个长尾商品曝光，全局品类覆盖率提升至72.24%，人均品类覆盖数达7.14 个，同品类规则合规率86.35%。
